\chapter{Requirements Traceability and Supporting Information}
\label{chap:traceability}

This chapter provides supporting information that complements the
Software Requirements Specification for the EcoTrace Intelligent
Electronic Waste Recovery and Circular Economy Management System.

The traceability matrices, data dictionary, database schema,
terminology, UML documentation, and supporting reference materials
presented in this chapter facilitate implementation, verification,
validation, maintenance, and future enhancement of the system.

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\section{Introduction}

Requirements traceability establishes relationships between the various
requirements defined throughout this Software Requirements
Specification.

Traceability supports impact analysis, verification, validation,
maintenance, and change management by ensuring that every requirement
can be linked to one or more system functions, business rules, use
cases, and database components.

\section{Firestore Security Rules Overview}

The EcoTrace platform shall implement Firestore Security Rules to
protect cloud-hosted information.

Major security principles include:

\begin{itemize}

\item Only authenticated users may access protected collections.

\item Users may only modify their own profile information.

\item Administrators possess elevated privileges.

\item Marketplace listings shall be editable only by their owners.

\item Audit logs shall only be accessible to administrators.

\item All write operations shall require authentication.

\item Cloud Storage uploads shall be authenticated.

\item Access permissions shall follow Role-Based Access Control.

\end{itemize}

\section{API Overview}
\label{sec:api-overview}

The EcoTrace platform adopts a service-oriented architecture in which
Flutter applications communicate with Firebase cloud services and
external APIs through secure HTTPS connections.

The major APIs and cloud services used by the platform are summarized
in Table~\ref{tab:api_overview}.

\\

\hline

\endfirsthead

\hline

\textbf{API / Service}

&

\textbf{Purpose}

\\

\hline

\endhead

Firebase Authentication

&

User authentication and identity management.

\\

\hline

The APIs shall exchange information using JSON over secure HTTPS
connections.

All communication involving authentication, user information,
electronic waste records, and AI predictions shall be encrypted using
Transport Layer Security (TLS).
